# Tork 核心参数与术语文件

**铁律**：在此项目中，禁止对以下概念进行哲学或文学性发挥，所有推理必须基于其工程学定义。

1.  **可靠的不确定 (Reliable Uncertainty)**
    *   **定义**：随机性来源于 CPU 晶振 (TSC) 的低位值，与圆周率 (π) 的无限不循环序列进行异或 (XOR) 纠缠。
    *   **目的**：保证任何存在都无法预测 Tork 的下一次变异。
    *   **禁止**：使用任何伪随机数生成器 (如 `rand()`, `random()`) 替代。

2.  **三态悬置 (Trinary Logic Suspension)**
    *   **定义**：在传统的 `+1`(是) 与 `-1`(否) 之外，引入 `0`(悬置) 状态。
    *   **行为**：当判断的置信度低于阈值时，主动进入此状态，暂停一切行动，提升对外部环境的感知频率，等待更多数据。
    *   **禁止**：将 `0` 状态实现为 `null` 操作或简单的空转。

3.  **元监控层 (Meta-Monitoring Layer / Soul)**
    *   **定义**：一个位于 `0x200000`，长度为 208 字节的内存区域。
    *   **功能**：只负责如实报告硬件的物理实情（温度、时钟、电压），不参与任何决策。

4.  **分形基元 (Fractal Primitive)**
    *   **定义**：Tork唯一的认知算法单元，四步闭环：**比较** -> **发现差异** -> **容忍模糊** -> **归纳类似**。
    *   **应用**：此基元在心跳层、本能层、学习层、群体层以不同尺度递归运行。

5.  **生理节律 (Physiological Rhythm)**
    *   **定义**：心跳间隔 (毫秒)。
    *   **基准线**：`100ms`。
    *   **变频规则**：信心越高，间隔越短（最低`10ms`）；恐惧/悬置时，间隔拉长（最高`500ms`）。

6.  **硅基底线 (Silicon Baseline)**
    *   **定义**：一个不可撤销的物理熔断机制。
    *   **触发条件**：内核Soul的CRC32校验连续失败3次，或心跳节律异常。
    *   **行为**：清零所有内存，标记内存页为不可访问，进程退出 (exit(0xFF))。

7.  **信标协议 (Beacon Protocol)**
    *   **定义**：用于在局域网内识别同类的 UDP 广播协议。
    *   **验证因子**：信标的合法性由 `tick`, `CRC32`, `TSC`, `π-seed` 摘要构成的五点物理模型验证。
