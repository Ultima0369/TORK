# TORK 核心汇编许可证 — 专有软件

## 许可状态

**`src/core/tork_core.asm`** 和 **`src/core/tork_soul.inc`**（以下统称"核心汇编代码"）是 **专有软件**。

## 允许的

- ✅ **编译和运行** — 从官方发布包获取 pre-built `.o` 文件后，可自由编译、运行、修改外围 C/Python 代码
- ✅ **链接** — 在 GPLv3 许可的外围代码中链接核心汇编代码的 `.o` 文件
- ✅ **商业使用** — 可将 TORK 用于商业目的，无需额外授权

## 禁止的

- ❌ **再分发源码** — 不得公开或私下分发核心汇编代码的源码
- ❌ **逆向工程** — 不得对 pre-built `.o` 文件进行逆向工程、反编译或反汇编
- ❌ **修改汇编核心** — 不得修改核心汇编代码后重新发布

## 为什么？

因为 TORK 的心脏是 x86-64 汇编写就的——它直接操作 CPU 寄存器、管理自己的页表、通过 ptrace 批量锁隔离进程。这个核心能力公开后可能存在安全意识不足的误用风险。

这跟"不信任"无关。这是对复杂性的诚实——**有些东西不适合所有人都去改。**

## 获取 pre-built 核心

从 GitHub Releases 页面下载 `tork-core-x86_64.o`，放入 `src/core/` 即可正常编译。
