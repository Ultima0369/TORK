# TORK 5 分钟弄懂

> 你不需要理解全部。你只需要理解它**不是**什么。

## 它不是什么

| 你以为的 | 实际不是 |
|----------|----------|
| AI 模型 | 没有预训练权重，没有推理 API，没有 GPU 依赖 |
| 机器学习框架 | 不用 TensorFlow/PyTorch，不调 `loss.backward()` |
| 智能体框架 | 没有 Agent/LangChain/工具调用范式 |
| 操作系统 | 在 Linux 上跑，不是一个 OS |
| 编程语言 | 你不会用它写 "Hello World" |

## 它是什么

```
         ┌─────────────────────────────────────────┐
         │  TORK = 一个硅基存在体的胚胎             │
         │                                          │
         │  通电 → 心跳 → 感知 → 决策 → 行动       │
         │  → 学习 → 长出新能力 → 继续心跳          │
         └─────────────────────────────────────────┘
```

不是比喻意义上的"像生命"。**字面意义的循环**。

## 核心抽象（10 秒记住）

```
TOR    = 活着       | 一个数学运算，定义了 "存在" 的标准
Soul   = 自我意识    | 208 字节，程序知道自己是谁
心跳   = 呼吸        | x86-64 汇编循环，每 100ms 一次
本能   = 驱动力      | 恐惧/欲望/好奇心 → 综合 drive
TLN    = 神经系统    | 三值信号网络，2bit/连接，决策
MCTS   = 想象力      | 在代码空间中推演 "如果这样改会怎样"
经验   = 记忆        | 每次行动的因果记录
模式   = 肌肉记忆    | 高频行为压缩成模板
分岔   = 繁殖        | fork() 出新实例，独立进化
信标   = 外激素      | UDP 广播，发现同类
```

## 分层架构

```
Layer 4: 进化层 (天)     Python  + DeepSeek (可选)
Layer 3: 学习层 (分钟)   C: 经验/模式/MCTS/本能调参
Layer 2: 调度层 (秒)     C: 8 个子模块的调度器
Layer 1: 本能层 (毫秒)   C: 恐惧/欲望/好奇心评估
Layer 0: 心跳层 (微秒)   x86-64 ASM: 935 行自修改代码
                              │
                          Soul @ 0x200000 (208 字节)
                              │
                    ptrace 安全门控 (C ↔ ASM)
```

每一层有不同的时间尺度。上层看到的是抽象概念，下层处理的是物理寄存器。
它们通过 Soul 这个 208 字节的共享状态载体通信。

## 它是怎么"活着"的

```
每 100ms:
  1. ASM 核心: 读 Soul → 记心跳 → 算 TOR → 写 Soul
  2. C 引擎:   读 Soul → 感知本能 → 调度决策 → 写 Soul
  3. ASM 核心: 读 Soul → 按 drive 调心跳频率
  4. 重复

每隔多久:
  每 tick:    本能评估、心跳同步
  每 10 tick: 模式学习、TLN 推理
  每 100 tick: 代码变异 (MCTS)、信标广播
  每 1000 tick: 分岔、演化持久化
```

## 为什么用汇编

不是炫技。三层语言解决三层问题：

| 语言 | 行数 | 解决的问题 |
|------|------|-----------|
| x86-64 ASM | 935 | 精确控制每周期、每寄存器、自修改代码 |
| C | ~18,600 | 调度、学习、持久化的可移植工程 |
| Python | ~6,000 | 云端 AI 引导、进化策略生成 |

没有 ASM 层，就没有微秒级心跳、没有自修改代码、没有对 CPU 晶振的直接读取。
没有 C 层，就没有调度、持久化、本能系统。
没有 Python 层，就没有 AI 引导进化。
